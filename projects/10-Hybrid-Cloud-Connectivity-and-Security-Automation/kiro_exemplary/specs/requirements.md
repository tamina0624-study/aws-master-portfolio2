# Requirements: ハイブリッド・クラウド接続とセキュリティ自動化

## 1. 概要

本ドキュメントは、オンプレミス環境とAWSクラウド間のハイブリッド接続およびセキュリティ自動化システムの要件を定義する。

## 2. ビジネス要件

### 2.1 目的
- オンプレミス環境からAWS上のデータベースに格納された機密情報へセキュアにアクセスする
- VPN接続の脆弱性を悪用した不正アクセスに対する自動防御を実現する

### 2.2 ステークホルダー
- インフラ管理者: システム構築・運用
- アプリケーション開発者: DB接続アプリケーションの利用
- セキュリティ担当者: 脅威検知・対応の監視

## 3. 機能要件

### FR-001: VPN接続
- [ ] オンプレミス環境とAWS VPC間をSite-to-Site VPNで接続する
- [ ] IPsec暗号化による通信の秘匿性を確保する
- [ ] BGPによる動的ルーティングを実装する
- [ ] 2本のトンネルによる経路冗長を確保する

### FR-002: アプリケーションサーバー
- [ ] AWS VPC内にFlask Webアプリケーションを稼働させる
- [ ] RDS (MySQL) への接続機能を提供する
- [ ] ヘルスチェックエンドポイントを実装する
- [ ] systemdによるプロセス管理で可用性を確保する

### FR-003: データベース
- [ ] MySQL RDSをプライベートサブネットに配置する
- [ ] IAM認証を有効化する
- [ ] AppServerからのみアクセス可能とする（SG制御）
- [ ] パブリックアクセスを無効化する

### FR-004: 脅威検知・自動防御
- [ ] GuardDutyによるリアルタイム脅威検知を有効化する
- [ ] EventBridgeで検知イベントをLambdaへ転送する
- [ ] Lambdaが攻撃元IPをWAF deny-ipsetに自動追加する
- [ ] 以下の脅威タイプを検知対象とする:
  - UnauthorizedAccess（不正アクセス）
  - Recon（偵察行為）
  - Backdoor（バックドア）
  - CryptoCurrency（マイニング）
  - Impact（破壊・改ざん）
  - Persistence（永続化）
  - PenTest（ペネトレーションテスト）
  - Behavior（異常行動）
  - Trojan（マルウェア）

### FR-005: アクセス制御
- [ ] WAF IPSetによるWeb層のIP制御を実装する
- [ ] セキュリティグループによるインスタンス単位の最小権限制御を実装する
- [ ] ネットワークACLによるサブネット単位の制御を実装する
- [ ] Web UIは許可IPリストからのみアクセス可能とする

## 4. 非機能要件

### NFR-001: セキュリティ
- 全リソース名の先頭に `xxxx-x` プレフィックスを付与する
- 同文字列 `xxxx-x` のタグも付与する
- クレデンシャルはSSM Parameter Storeで管理する（ハードコード禁止）
- セキュリティグループで All Traffic / All Protocol を使用しない
- `0.0.0.0/0` port 22, port 445 のSGルールを作成しない
- IAMロールは最小権限を適用する（AdministratorAccess禁止）
- S3バケットのパブリックアクセスを制限する

### NFR-002: リージョン・コスト
- デプロイリージョンは `us-east-1` のみ
- 必要最小限のコスト構成とする（t2.micro, db.t4g.micro等）

### NFR-003: 既存リソースとの共存
- 既存AWSリソースと競合しない
- 既存AWSリソースを削除・変更しない

### NFR-004: 可用性
- EC2管理はSSM Session Manager経由で行う
- アプリケーション通信を阻害しない管理経路を確保する

### NFR-005: 運用性
- Terraformによるインフラのコード管理（IaC）
- モジュール分割による再利用性の確保
- タグによるコスト管理・リソース識別

## 5. 制約事項

- AWSアカウントの権限制約に従う
- VPN接続はAWS Site-to-Site VPN（ipsec.1）を使用する
- オンプレミス環境はAWS上の別VPCで疑似的に構築する
- Lambda関数のランタイムはPython 3.12を使用する

## 6. 受入基準

- [ ] VPNトンネルが2本ともUP状態であること
- [ ] オンプレ側EC2からAWS側RDSへの接続が成功すること
- [ ] GuardDutyが有効化されていること
- [ ] テスト用の脅威シミュレーションでLambdaが発火し、deny-ipsetが更新されること
- [ ] 許可IPリスト外からのWeb UIアクセスが拒否されること
- [ ] 全リソースに `xxxx-x` プレフィックスとタグが付与されていること
